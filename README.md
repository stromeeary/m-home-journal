# m-home-journal

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，便于直接通过浏览器访问。仓库本身不绑定任何特定域名，也不针对具体网站或服务。

## 目录说明

```
m-home-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（如样式表、图片、脚本等）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名即页面标识。
- `assets/`：按需存放页面所需的公共或私有资源，建议按功能子目录整理。

## 页面归档说明

- 所有页面均为独立 HTML 文件，不依赖后端服务。
- 页面之间无强制关联，可单独访问或引用。
- 归档内容以静态网页形式保存，便于长期留存与分发。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在 `assets/` 中添加所需资源。
- 更新页面：直接修改对应 HTML 或资源文件，保持目录结构清晰。
- 删除页面：移除相关文件及引用资源，确保仓库整洁。
- 建议在提交信息中简要说明变更内容，方便追溯历史。
